◆ allow_reconnect

bool trpc::ServiceProxyOption::allow_reconnect {kDefaultAllowReconnect}

Whether to support reconnection in fixed connection mode, the default value is true.

For scenarios where reconnection is not allowed, such as transactional operations, set this value to false.

◆ callee_name

std::string trpc::ServiceProxyOption::callee_name

The name of callee service, used for metrics and tracing report.

If empty, it will be initialized to the name of service proxy.

◆ connect_timeout

uint32_t trpc::ServiceProxyOption::connect_timeout {kDefaultConnectTimeout}

The duration of connection timeout in milliseconds.

Zero means no connection timeout checking. Note: it's supported only in IO/Handle separation and merging mode.

◆ is_reconnection

bool trpc::ServiceProxyOption::is_reconnection {kDefaultIsReconnection}

Whether to reconnect after the idle connection is disconnected when reach connection idle timeout.

It's used in connection complex under IO/Handle separation and merging mode.

◆ request_timeout_check_interval

uint32_t trpc::ServiceProxyOption::request_timeout_check_interval {kDefaultRequestTimeoutCheckInterval}

The interval for request timeout detection, in milliseconds.

It's used in IO/Handle separation and merging mode.

◆ stream_max_window_size

uint32_t trpc::ServiceProxyOption::stream_max_window_size {kDefaultStreamMaxWindowSize}

The size of the sliding window for flow control, in bytes.

The default value is 65535, and a value of 0 means that flow control is disabled. Currently, flow control is effective for tRPC streaming.
