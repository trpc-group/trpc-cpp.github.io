Unary synchronous call with NoncontiguousBuffer input parameter and pb output parameter.

Parameters
[in]reqThe NoncontiguousBuffer after PB serialization.
[out]rspThe pb message.
Note
Use case: It is desirable to reuse the same request data structure each time when make a request, while only modifying certain fields (thus avoiding duplicate copying). In this case, it is allowed to pass in a serialized buffer as the request, so that the structure can be modified during the call. Then call this interface after serializing the structure into binary data.
