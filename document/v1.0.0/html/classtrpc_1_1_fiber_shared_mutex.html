Detailed Description

Analogous to std::shared_mutex, but it for fiber.

Note
Performance-wise, reader-writer lock does NOT perform well unless your critical section is sufficient large. In certain cases, reader-writer lock can perform worse than Mutex. If reader performance is critical to you, consider using other methods (e.g., thread-local cache, hazard pointers, FiberSeqLock, ...).
