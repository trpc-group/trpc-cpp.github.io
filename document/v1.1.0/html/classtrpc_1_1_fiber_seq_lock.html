Detailed Description

Reader/writer consistent mechanism without starving writers.

This type of lock for data where the reader wants a consistent set of information and is willing to retry if the information changes. Readers never block but they may have to retry if a writer is in progress. Writers do not wait for readers. This will not work for data that contains pointers, because any writer could invalidate a pointer that a reader was following. Read first: https://en.wikipedia.org/wiki/Seqlock

Reader usage:
FiberSeqLoc seqlock;
do {
seq = seqlock.ReadSeqBegin();
...
} while (seqlock.ReadSeqRetry(seq));
Writer usage:
seqlock.WriteSeqLock();
...
seqlock.WriteSeqUnlock();
