◆ execution_context

FiberExecutionContext* trpc::Fiber::Attributes::execution_context = nullptr

If set, fiber's start procedure is run in this execution context.

Fiber will take a reference to the execution context, so you're safe to release your own ref. once Fiber is constructed.

◆ is_fiber_reactor

bool trpc::Fiber::Attributes::is_fiber_reactor = false

If Set, it is a reactor fiber.

Note
Only used inside the framework

◆ scheduling_group

std::size_t trpc::Fiber::Attributes::scheduling_group = kNearestSchedulingGroup

Which scheduling group should the fiber be initially placed in.

Note that unless you also have scheduling_group_local set, the fiber can be stolen by workers belonging to other scheduling group.

◆ scheduling_group_local

bool trpc::Fiber::Attributes::scheduling_group_local = false

If set, scheduling_group is enforced.

(i.e., work-stealing is disabled on this fiber.)
